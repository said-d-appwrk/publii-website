Running ExcelPipe

ExcelPipe has two very important pre-requisites when running



Microsoft Excel is not already running (especially with open documents).
No documents are opened by someone else (no sharing)


Both of these situations lead to file sharing violations. ExcelPipe may appear to hang, and you will need to Alt-Tab to Microsoft Excel to close the 'Open Read Only' dialog box.



This leads to our two recommendations when running ExcelPipe:

Please close ALL other applications. This ensures that no programs have an Excel document open.



If ExcelPipe says that Microsoft Excel is still running, you may have to press Ctrl+Alt+Del to access the Task Manager and kill of all instances of EXCEL.EXE manually.

If you are running ExcelPipe across a network, file share or remote machine of any description, you MUST ensure that ExcelPipe has exclusive access to those files, either through running ExcelPipe after hours if you are guaranteed to have exclusive access, or by denying all users access to the files while ExcelPipe is being run.



Speed and Performance

To get the fastest speed out of ExcelPipe:



Ensure you disable virus scanning so that Microsoft Excel will open documents faster

Close all other running applications

If possible, run ExcelPipe directly on your File Server (Server Edition required, and local install of MS Office)

Increase the process priority of EXCEL.EXE in task manager.
